# 构建分析报告

每次生产构建完成后，Lite Vite 会自动在输出目录生成一份 `build-report.html` 可视化分析报告，帮助你了解和优化产物质量。

## 查看报告

构建完成后，直接在浏览器中打开报告文件：

```bash
lite-vite build
# ✓ Built in 2.35s
# Output: dist
# 📊 构建报告: dist/build-report.html

open dist/build-report.html
```

## 报告内容

### 总览面板

报告顶部展示关键指标：

| 指标 | 说明 |
|---|---|
| 构建时间 | 从开始到完成的总耗时 |
| 总体积 | 所有产物文件的体积总和 |
| 文件数量 | 产物文件的总数 |
| 模块数量 | 参与打包的源码模块总数 |
| 入口文件 | 构建使用的入口 |
| 输出格式 | ESM / CJS |

### 体积分类统计

将产物按文件类型分类，展示各类型的体积占比：

- **JavaScript** —— `.js` / `.mjs` / `.cjs` 文件
- **CSS** —— `.css` 文件
- **图片** —— `.png` / `.jpg` / `.jpeg` / `.gif` / `.webp` / `.avif`
- **SVG** —— `.svg` 文件
- **字体** —— `.woff` / `.woff2` / `.ttf`
- **HTML** —— `.html` 文件
- **Sourcemap** —— `.map` 文件
- **其他** —— 未分类文件

### 文件清单

列出每个产物文件的详细信息：

| 列 | 说明 |
|---|---|
| 文件名 | 产物文件名 |
| 类型 | chunk（代码块）/ asset（资源文件） |
| 原始体积 | 文件实际大小 |
| gzip 估算 | 预估 gzip 压缩后体积（约原始体积的 35%） |
| brotli 估算 | 预估 brotli 压缩后体积（约原始体积的 25%） |

### 模块分析

展示每个 chunk 中包含的源码模块：

- 模块名称（相对于项目根目录的路径）
- 模块在最终产物中的渲染体积
- chunk 的导入/导出关系

### 重复依赖检测

自动扫描是否有第三方包被多个 chunk 重复打包：

```
⚠️ 检测到 2 个重复依赖

lodash
  ├── node_modules/lodash (chunk main.js) - 12.3 KB
  └── node_modules/lodash (chunk vendor.js) - 12.3 KB
  总浪费: 12.3 KB
```

### 网络模拟

估算不同网络环境下的首屏加载时间：

| 网络环境 | 速度 |
|---|---|
| 5G | 100 Mbps |
| 4G | 10 Mbps |
| 3G | 1.5 Mbps |
| 慢速 3G | 400 Kbps |
| 2G | 50 Kbps |

### 构建历史对比

报告会记录每次构建的数据，支持：

- **历史趋势** —— 查看总体积、JS 体积、CSS 体积随时间的变化
- **版本对比** —— 对比两次构建之间的文件差异（新增/删除/变更/未变化）
- **增量分析** —— 精确展示每个文件的体积变化量

历史数据保存在 `.lite-vite/build-history.json` 中，最多保留 50 条记录。

### 优化建议

报告基于分析结果提供自动化建议：

| 条件 | 建议 |
|---|---|
| JS 体积 > 500 KB | 建议使用代码分割或懒加载减小首屏加载体积 |
| 所有 JS 在单个 chunk | 建议将第三方库拆分为独立 chunk |
| CSS 体积为 0 | 请确认样式是否正确提取 |
| 包含 Sourcemap | 确保不要将其公开部署到 CDN |
| 存在重复依赖 | 提示浪费的体积 |
| 总体积 > 2 MB | 建议启用压缩、tree-shaking |
| 大图片（> 200 KB）| 建议使用 WebP/AVIF 格式 |
| 无明显问题 | "构建状态良好，未发现明显问题" |

## 历史数据管理

### 存储位置

```
项目根目录/
└── .lite-vite/
    └── build-history.json    # 构建历史数据
```

### 清除历史

```bash
rm -rf .lite-vite/build-history.json
```

下次构建会重新创建。
